# 藤崎 Google Drive フォルダ整理資料

作成日：2026-04-02
対象：GoosePath × 藤崎 共有ドライブ

---

## 1. 現状の課題（サマリー）

- 定例MTG資料が週次フォルダとして **38件以上** フラットに並んでいる
- キャンペーンフォルダが体系なくルート直下に散在
- 命名規則がバラバラ（区切り文字・日付形式・全角半角が混在）
- フォルダの役割・用途が不明確なものが多い
- 新設フォルダと旧フォルダが並存し、どちらが正しいか不明

→ **旧コンテンツはすべて `_archive` フォルダに移動し、ゼロベースで再構築する。**

---

## 2. 設計方針

| 原則 | 内容 |
|------|------|
| **用途ベース** | 「誰が何のために使うか」でフォルダを分類する |
| **時系列はフォルダ内で管理** | ルートには時系列フォルダを置かない |
| **命名規則を統一** | `YYYYMMDD_` プレフィックス ＋ 半角アンダーバー区切り |
| **階層は最大3層** | それ以上深くしない |
| **アーカイブを明示** | 過去データは `_archive` に集約し、現役フォルダと混在させない |

## 4. 各フォルダの用途定義

### 01_定例MTG
毎週の定例ミーティングで共有するデータ・資料を格納。年→月の2階層で管理。
- 各月フォルダの中に `YYYYMMDD_GoosePath共有資料` を置く
- **入れるもの**：週次共有データ、議事録、アジェンダ

### 02_データ
藤崎からGoosePathへ共有されるマスタ・ローデータを格納。
- **入れるもの**：受注CSV、会員リスト、メール行動ログ、店舗マスタ
- データは上書き更新 or 日付付きファイル名で管理

### 03_分析・レポート
GoosePathが作成した分析ドキュメント・レポートを格納。
- **入れるもの**：ファネル分析、KPIサマリー、施策効果レポート

### 04_キャンペーン
イベント・季節キャンペーンごとのフォルダ。年→キャンペーン名の2階層で管理。
- **入れるもの**：施策資料、LP素材、数値データ、振り返り

### 05_サイト改善
サイト全体の改善提案・LP制作に関わる資料を格納。
- **入れるもの**：改善提案書、ワイヤーフレーム、LP入稿資料

### _archive
整理以前の旧フォルダ群。参照のみ・原則編集しない。

## 5. ファイル命名規則

### 基本フォーマット

```
YYYYMMDD_[種別]_[対象期間].[拡張子]
```

| 要素 | 内容 | 備考 |
|------|------|------|
| `YYYYMMDD` | **受領日**（ファイルを受け取った日） | 例：`20260402` |
| `[種別]` | データの種類（下表参照） | 半角アンダーバー区切り |
| `[対象期間]` | データが対象とする期間 | 月次・週次で形式を変える |

---

### 種別コード

| データ種別 | コード | 例 |
|-----------|--------|-----|
| EC受注データ | `EC受注` | `20260402_EC受注_2026-03.csv` |
| 会員データ | `会員` | `20260402_会員_2026-03.csv` |
| メール行動ログ | `メールログ` | `20260402_メールログ_2026-03.csv` |
| 店舗取引データ | `店舗取引` | `20260402_店舗取引_2026-03.csv` |

---

### 対象期間の書き方

**月次データ**
```
YYYY-MM
例：2026-03
```

**週次データ**
```
YYYY-MM-DD~MM-DD（週の月曜〜日曜）
例：2026-03-23~03-29
```

**累計・範囲指定**
```
YYYY-MM~YYYY-MM
例：2025-10~2026-03
```

---

### 命名例

```
20260402_EC受注_2026-03.csv                  ← 3月分の受注データ（月次）
20260402_EC受注_2026-03-23~03-29.csv         ← 先週分の受注データ（週次）
20260402_会員_2026-03.csv                    ← 3月分の会員データ
20260402_メールログ_2026-03.csv              ← 3月分のメール行動ログ
20260402_店舗取引_2026-03.csv                ← 3月分の店舗取引データ
20260402_EC受注_2025-10~2026-03.csv          ← 半期累計の受注データ
```

---

### NG例（今後使わない）

```
受注データ_3月.csv          → 受領日なし・種別コード不統一
20260402受注.csv            → アンダーバー区切りなし
2026-03_EC受注データ.csv    → 受領日でなく対象期間が先頭
注文履歴（1）.xlsx          → 括弧・日本語のみ・連番管理
```
