## URL命名規則の分析

### カテゴリーID体系

#### 基本形式
```
[大カテゴリー]-[連番]
例: food-01, food-02, life-01
```

#### 特集形式
```
[大カテゴリー]-[テーマ名]
例: food-winesale, food-fruit
```

#### 横断カテゴリー形式
```
[親カテゴリー]-[分類]-[詳細分類]-[連番]
例: life-B_W-item-food_drink-01
```

#### キャンペーン形式
```
[キャンペーン名]-[種別]-[連番]
例: summer_gift-item-02
```

#### 階層構造形式
```
[大カテゴリー]-[中カテゴリー]-[小カテゴリー]
例: food-11-03
```

### 商品コード体系

#### パターン1: 英数字混合
```
fd25050004  (fd + 数字8桁)
```

#### パターン2: 英数字-ハイフン区切り
```
25s-01013016  (数字+英字-数字8桁)
25s-01009019
25s-01124019
```

### ニュース連番体系
```
10000000番台の連番
10000105, 10000104, 10000103...
```

## SEO・技術的特徴

### URL構造の利点
1. **階層的**: カテゴリーが明確に分類されている
2. **予測可能**: 命名規則が一貫している
3. **拡張性**: 新カテゴリー追加が容易

### 改善の余地
1. **深い階層**: 一部のURLが長くなる傾向
2. **パラメータ依存**: 静的URLでない部分が多い
3. **日本語非対応**: URLに日本語が含まれていない
