# 春の北海道展 2026｜情報設計・導線設計アウトライン

## 1. 全体の導線構造

ユーザーが北海道展に関わる情報に触れるタッチポイントは大きく3つ。

```
[公式サイト催事ページ]  →  [EC LP（藤崎オンライン）]  →  [EC商品ページ → 購入]
     ↑                         ↑
  店頭来場の情報収集        オンライン購入
  が主目的の人が多い        が目的の人はごく少数
```

**前年の問題：** この3つが一本道でつながっていた。催事ページの最下部に「藤崎オンラインへ」のリンクがあるだけ。店頭情報を見た流れでECに遷移するため、「店頭商品がオンラインで買える」と誤認した流入が大半。結果、EC LP転換率0.37%。

**今年の設計方針：** 公式サイト催事ページに2つの役割を持たせる。

| 役割 | 対象 | ゴール |
|---|---|---|
| ① 店頭来場者向けの催事情報 | 会場に来る人 | 何があるか把握して来場 |
| ② EC送客の起点 | 会場に来られない人 / 来るけどオンラインにも興味ある人 | 正しい期待値でEC LPへ遷移 |

この2つを**混ぜない**ことが最重要。

## 2. 公式サイト催事ページの情報設計

### 2-1. ページ構成案

```
A. メインビジュアル + タイトル + 開催情報
B. カテゴリナビ（ページ内リンク）
C. EC送客ブロック（上部）      ← 新設
D. 店頭商品セクション（カテゴリ別）
E. WEBチラシ
F. EC送客ブロック（下部）      ← 改善
G. 注意事項
```

### 2-2. 各ブロックの役割と設計意図

**A. メインビジュアル + 開催情報**
- 変更なし。日程・場所・キャッチコピー。
- ここでの訴求は催事全体の期待感。ECの話はしない。

**B. カテゴリナビ**
- 店頭カテゴリのページ内リンク（スイーツ / グルメ / 海鮮弁当 / ワイン・ビール等）
- ここに「オンラインストア」ボタンを追加するかどうかは要検討。
  - **入れる場合：** 色を変えて差別化し、ECセクションへのページ内リンク。視認性は上がる。
  - **入れない場合：** 店頭情報を見に来た人の導線を邪魔しない。ECはECセクションで完結。
  - → 判断ポイント：カテゴリナビの目的は「店頭で何があるか素早く見る」こと。EC混在はノイズになりうる。

**C. EC送客ブロック（上部） ← 新設・今回の最重要改善**
- **位置：** カテゴリナビと店頭商品セクションの間。
- **意図：** 店頭情報を見始める前に、ECの存在と「店頭とは別のラインナップであること」を伝える。ここで興味を持った人はECに遷移し、興味がない人はスクロールして店頭情報を見る。
- **設計の要点：**
  - 店頭セクションとは視覚的に明確に区別する（色・囲みで差別化）
  - 「店頭催事とは異なるラインナップ」を明示する（期待値コントロール）
  - ECにどんな商品があるかの具体性を出す（カテゴリ名・商品数）
  - 「産地直送・送料込・ご自宅にお届け」はECの利点として添える
  - CTAボタンでEC LPへ遷移
- **文言の方向性：**
  - NG例：「おうちに北海道のグルメが届く」← 何が届くかわからない、店頭と同じと誤認する
  - OK例：「会場とは別のラインナップ。オンライン限定の北海道産直グルメ、約40商品」← 違いが明確

**D. 店頭商品セクション**
- ここは店頭情報であり、今回の改善の主対象ではない。
- ただし、情報設計として以下は検討に値する：
  - 前半/後半の期間限定商品の扱い（来場時期の判断に影響する情報）
  - 数量限定の表記（前年踏襲で赤字表示）

**E. WEBチラシ**
- 変更なし。

**F. EC送客ブロック（下部） ← 改善**
- **位置：** WEBチラシの後。前年と同じ位置だが、文言を改善。
- **意図：** ページを最後まで見た人への再アプローチ。上部で見逃した人のキャッチアップ。
- **前年からの改善：**
  - 「おうちに北海道のグルメが届く」→「会場では買えない、オンライン限定の北海道産直グルメ」
  - 注釈で「※オンラインストアの商品は、店頭催事とは異なるラインナップです」を明記

**G. 注意事項**
- 前年踏襲。

## 3. EC送客の訴求方針

### 3-1. やるべきこと

| # | 訴求ポイント | 理由 |
|---|---|---|
| 1 | **店頭とは別の商品であることの明示** | 前年最大の離脱原因。誤認を防ぐことが最優先 |
| 2 | **ECにある商品の具体性** | 「北海道グルメ」だけでは弱い。「海鮮・お肉・スイーツなど約40商品」のように具体化 |
| 3 | **ECならではの利点** | 産地直送・送料込・ご自宅にお届け。来場できない人への明確なメリット |
| 4 | **体験としての訴求** | 「北海道の海鮮丼を自宅で」のような体験想起。ただし公式サイト側では簡潔に |

### 3-2. やらないこと

| # | やらないこと | 理由 |
|---|---|---|
| 1 | 店頭との比較・優位性の主張 | 店頭のほうが品数も体験も上。比較すると目劣りする |
| 2 | EC商品の詳細情報をこのページに載せる | ここは催事ページ。ECの詳細はEC LPの役割 |
| 3 | 過剰な装飾・バナー | MTの基本エディタで実装可能な範囲。トーンも維持 |

### 3-3. トーン

- 催事ページ全体のトーンを壊さない。催事の一部としてECが自然に紹介される形。
- 「売り込み」ではなく「ご案内」。「こちらもございます」のスタンス。

## 4. EC LP側の設計方針（別途詳細化）

公式サイトからの送客先であるEC LPも改善対象。ここでは方針のみ整理。

### 4-1. EC LPの役割

公式サイトから遷移してきた人に、**商品の魅力を伝えて「買いたい」を作る**こと。

前年の問題：商品一覧が並んでいるだけで、訴求がなかった。

### 4-2. EC LPで必要な要素（方針）

- **ファーストビューで「ここはオンライン限定の北海道産直グルメのページ」と伝える**
- **商品を体験として訴求する**（単なるスペック一覧ではなく）
- **カテゴリ分類で商品との出会いを最適化する**（約40商品を整理）
- **承り期間の明示**（期間限定の購買動機）

※EC LP側の具体的な設計は、公式サイト側の方針が固まった後に詳細化する。

## 5. 効果測定の設計

### 5-1. KPI（再掲）

| 指標 | 前年実績 | 目標 |
|---|---|---|
| EC LP転換率 | 0.37% | 1.0% |
| EC売上 | 約20万円 | 35万円 |

### 5-2. 計測すべきポイント

| 計測対象 | 意味 | 方法 |
|---|---|---|
| 催事ページ → EC LPのクリック率 | EC送客ブロックの効果 | GTMイベント or UTMパラメータ |
| 上部EC送客ブロック vs 下部のクリック比率 | どちらが効いているか | 別UTMパラメータで区別 |
| EC LP → 商品ページ遷移率 | LP上での訴求効果 | GA4 |
| EC LP転換率 | 最終的な成果 | GA4 |

### 5-3. UTMパラメータ案

- 上部EC送客ブロック：`?utm_source=fujisaki_event&utm_medium=referral&utm_campaign=hokkaido2026&utm_content=upper_banner`
- 下部EC送客ブロック：`?utm_source=fujisaki_event&utm_medium=referral&utm_campaign=hokkaido2026&utm_content=lower_banner`
- カテゴリナビ内ボタン（入れる場合）：`?utm_source=fujisaki_event&utm_medium=referral&utm_campaign=hokkaido2026&utm_content=category_nav`

| # | 項目 | 内容 |

| 1 | カテゴリナビに「オンラインストア」ボタンを入れるか | 店頭情報の導線にECを混在させるかどうか |
